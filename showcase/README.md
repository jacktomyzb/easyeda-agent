# Showcase — 用户最佳实现板提交板块

用 easyeda-agent 做出来的**真实板子**都可以提交到这里。每个案例一个目录,带版本号,
沉淀从需求到成品的完整决策链——这既是作品墙,也是 agent 的回归语料库。

## 目录与版本规范

```
showcase/
├── README.md                     # 本文
└── NNN-<板名>/                   # 案例编号(001 起)+ 短横线板名
    ├── case.md                   # 案例卡:一句话需求、来源、作者、状态、版本历史
    ├── s0-proposal.md            # S0 设计方案书(决策点 + 用户拍板记录)
    ├── spec.json                 # 方案书 spec(机器可读,后续阶段引用)
    └── vX.Y/                     # 每个设计版本一个目录
        ├── snapshot-pcb.png      # 原生截图(stage-snapshot 抓取,非数据渲染图)
        ├── data/                 # drc/nets/tracks 等 json 数据包
        └── notes.md              # 本版改动、偏差、遗留问题
```

**版本号**:`vX.Y` —— X 递增 = 布局/架构级重做,Y 递增 = 同架构修线/微调。
首版从 `v0.1` 起,打样验证通过后升 `v1.0`。

## 提交要求(硬门)

1. **原始需求不加工**:case.md 里写客户口吻的需求原文,不喂 BOM/网表。
2. **决策留痕**:s0-proposal.md 记录每个决策点的选项与最终拍板(谁选的、为什么)。
3. **数据佐证**:DRC/check/网表数据包入 `data/`,截图必须是原生截图(stage-snapshot 把关)。
4. **偏差诚实**:降级/妥协(如 USB 单取向)必须写进 notes.md,不许静默。
